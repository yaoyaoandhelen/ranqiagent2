# 重庆社科科学院发展研究奖评审结果分析 PRD

## 1. 项目背景

重庆社科科学院发展研究奖评审结果分析看板用于汇总参评成果、获奖结果、单位表现、成果形式和所属领域分布，辅助管理人员快速了解本次评审结果结构，识别获奖集中度、成果类型分布和单位贡献情况。

当前版本为静态 HTML 看板，部署于 GitHub Pages。

线上地址：https://yaoyaoandhelen.github.io/ranqiagent2/

## 2. 产品目标

1. 将专家评审后的奖项结果可视化，形成直观的后台分析页面。
2. 展示本次评审的总体规模、获奖规模、未获奖规模和奖项分布。
3. 按第一完成单位、成果形式、所属领域统计获奖数量排名。
4. 保持白底后台风格，页面结构简洁，适合汇报、截图和后续系统集成。

## 3. 用户角色

主要用户：社科院发展研究奖评审管理人员、系统运营人员、成果评奖业务负责人。

使用场景：
- 评审结束后查看本次奖项结构。
- 对外或内部汇报获奖数量、占比和排名。
- 发现获奖成果在单位、形式、领域上的分布特点。

## 4. 数据来源

### 4.1 参评成果基础表

文件：`成果导入.xlsx`

用途：提供参评成果基础信息，用于单位、成果形式、所属领域等维度统计。

主要字段：
- `成果编号`：成果唯一标识，导入后统一命名为 `编号`。
- `成果名称`：成果标题。
- `第一完成单位`：单位获奖排名统计字段。
- `成果形式`：成果形式获奖排名统计字段。
- `所属领域`：所属领域获奖排名统计字段。
- `申报主体类别`：保留字段，当前看板未展示。
- `成果分组结果`：保留字段，当前看板未展示。

### 4.2 奖项结果表

文件：
- `研究成果_评审结果_一等奖.xlsx`
- `研究成果_评审结果_二等奖.xlsx`
- `研究成果_评审结果_三等奖.xlsx`

用途：提供最终获奖名单和评分结果。

主要字段：
- `编号`：与成果导入表关联。
- `成果名称`：获奖成果名称。
- `小组评分`：专家小组评分。
- `大会评分`：大会评分。
- `综合最终得分`：最终综合得分。

系统在读取不同奖项文件时，根据文件来源补充 `奖项` 字段：
- 一等奖文件补充为 `一等奖`
- 二等奖文件补充为 `二等奖`
- 三等奖文件补充为 `三等奖`

### 4.3 业务假设

本次评审作品总量按业务假设设置为 `120` 份。

特等奖本次未产生，数量为 `0`。

未获奖数量计算方式：

```text
未获奖数量 = 评审作品总量 - 特等奖数量 - 一等奖数量 - 二等奖数量 - 三等奖数量
```

当前数据：
- 评审作品总量：120 份
- 特等奖：0 份
- 一等奖：24 份
- 二等奖：45 份
- 三等奖：30 份
- 未获奖：21 份
- 获奖总量：99 份
- 总体获奖率：82.5%

## 6. 功能需求与图表数据来源

### 6.1 页面标题与奖项颜色标签

展示内容：
- 页面标题：`重庆社科科学院发展研究奖评审结果分析`
- 奖项标签：特等奖、一等奖、二等奖、三等奖、未获奖

数据来源：
- 固定奖项枚举：`特等奖`、`一等奖`、`二等奖`、`三等奖`、`未获奖`
- 颜色配置来自前端 `awardColors`

展示规则：
- 标签用于统一解释页面内各奖项颜色。
- 不展示额外说明文字，保持页面简洁。

### 6.2 核心指标卡

展示指标：
- `评审作品总量`
- `获奖总量`
- `未获奖数量`

数据来源：
- `评审作品总量`：业务假设 `TOTAL_WORKS_ASSUMPTION = 120`
- `导入表可追踪数量`：成果导入表读取行数，当前为 `105`
- `获奖总量`：特等奖、一等奖、二等奖、三等奖数量之和，当前为 `99`
- `未获奖数量`：120 - 99 = `21`

计算口径：

```text
获奖总量 = 特等奖数量 + 一等奖数量 + 二等奖数量 + 三等奖数量
总体获奖率 = 获奖总量 / 评审作品总量
未获奖占比 = 未获奖数量 / 评审作品总量
```

### 6.3 奖项分布

展示形式：
- 4 个彩色卡片，分别展示特等奖、一等奖、二等奖、三等奖。

数据来源：
- `awardSummary`
- 特等奖数量来自业务设定，当前为 `0`
- 一等奖、二等奖、三等奖数量来自对应奖项结果表的记录数

展示字段：
- 奖项名称
- 奖项数量
- 奖项占比

计算口径：

```text
奖项占比 = 该奖项数量 / 评审作品总量 120
```

当前结果：
- 特等奖：0 份，0.0%
- 一等奖：24 份，20.0%
- 二等奖：45 份，37.5%
- 三等奖：30 份，25.0%

### 6.4 奖项占比与数量环形图

展示形式：
- 环形图 + 右侧奖项列表。
- 中心展示评审总量。

数据来源：
- `awardSummary`

包含奖项：
- 特等奖
- 一等奖
- 二等奖
- 三等奖
- 未获奖

计算口径：

```text
环形图总量 = 特等奖 + 一等奖 + 二等奖 + 三等奖 + 未获奖
奖项占比 = 该奖项数量 / 环形图总量
```

当前环形图总量为 `120` 份。

展示字段：
- 奖项名称
- 数量
- 占比

### 6.5 奖项结构环形图

展示形式：
- 环形图 + 获奖率重点数字。
- 图中标注各奖项占比。

数据来源：
- `awardSummary`
- `meta.awardedTotal`
- `meta.totalAssumption`

计算口径：

```text
奖项结构总量 = 特等奖 + 一等奖 + 二等奖 + 三等奖 + 未获奖 = 120
获奖成果 = 特等奖 + 一等奖 + 二等奖 + 三等奖 = 99
获奖率 = 获奖成果 / 评审作品总量
```

当前结果：
- 获奖成果：99 份
- 获奖率：82.5%
- 总基数：120 份

展示规则：
- 特等奖为 0 时，不在环形图扇区中占实际面积。
- 保留“特等奖本次未产生”的业务提示。

### 6.6 单位获奖数量排名

展示形式：
- 横向堆叠条形图。
- 每行展示一个第一完成单位。
- 右侧显示 `获奖数/申报数`。

数据来源：
- 基础维度字段：成果导入表 `第一完成单位`
- 奖项字段：由成果导入表与奖项结果表按 `编号` 关联后生成

统计逻辑：
1. 将成果导入表与获奖结果表通过 `编号` 左连接。
2. 未匹配到奖项的成果标记为 `未获奖`。
3. 按 `第一完成单位` 分组。
4. 统计各单位下各奖项数量。
5. 计算申报数、获奖数、获奖率。
6. 按获奖数、申报数、获奖率降序排序。

计算口径：

```text
申报数 = 该单位在成果导入表中的成果记录数
获奖数 = 特等奖 + 一等奖 + 二等奖 + 三等奖
获奖率 = 获奖数 / 申报数
```

展示范围：
- 当前展示前 14 个单位。

### 6.7 成果形式获奖排名

展示形式：
- 横向堆叠条形图。
- 每行展示一种成果形式。
- 右侧显示 `获奖数/申报数`。

数据来源：
- 基础维度字段：成果导入表 `成果形式`
- 奖项字段：由成果导入表与奖项结果表按 `编号` 关联后生成

数据清洗规则：
- 标准成果形式包括：`研究报告`、`政策建议`、`调研报告`、`著作`、`论文`
- 不在标准集合内的成果形式统一归类为 `待核验`

统计逻辑：
1. 按 `成果形式` 分组。
2. 统计各成果形式下特等奖、一等奖、二等奖、三等奖、未获奖数量。
3. 计算申报数、获奖数、获奖率。
4. 按获奖数、申报数、获奖率降序排序。

计算口径：

```text
申报数 = 该成果形式记录数
获奖数 = 特等奖 + 一等奖 + 二等奖 + 三等奖
获奖率 = 获奖数 / 申报数
```

### 6.8 所属领域获奖排名

展示形式：
- 横向堆叠条形图。
- 每行展示一个所属领域。
- 右侧显示 `获奖数/申报数`。

数据来源：
- 基础维度字段：成果导入表 `所属领域`
- 奖项字段：由成果导入表与奖项结果表按 `编号` 关联后生成

统计逻辑：
1. 按 `所属领域` 分组。
2. 统计各领域下特等奖、一等奖、二等奖、三等奖、未获奖数量。
3. 计算申报数、获奖数、获奖率。
4. 按获奖数、申报数、获奖率降序排序。

计算口径：

```text
申报数 = 该领域记录数
获奖数 = 特等奖 + 一等奖 + 二等奖 + 三等奖
获奖率 = 获奖数 / 申报数
```

## 7. 数据处理流程

1. 读取 `成果导入.xlsx`。
2. 标准化字段名称，将 `成果编号` 重命名为 `编号`。
3. 清洗文本字段，去除空值和首尾空格。
4. 读取一等奖、二等奖、三等奖评审结果表。
5. 根据文件来源补充 `奖项` 字段。
6. 合并三个奖项文件形成获奖结果表。
7. 将成果导入表与获奖结果表按 `编号` 左连接。
8. 未匹配获奖结果的成果标记为 `未获奖`。
9. 生成 `awardSummary`、`unitStats`、`formStats`、`fieldStats` 等前端渲染数据。
10. 将数据嵌入静态 HTML 页面。

## 9. 当前不展示内容

`成果名称获奖矩形树图` 模块已根据需求删除，不再在当前版本页面展示。

历史上该模块使用 `matrixRows` 数据，数据来源为成果导入表与奖项结果表按 `编号` 合并后，再按 `成果名称` 聚合。当前版本保留部分生成函数代码，但页面不再调用，也不作为当前看板功能范围。
